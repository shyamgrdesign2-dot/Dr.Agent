CHAPTER	01
The	Core	Idea	—	Why	This	Agent	Is	Different
Every	EMR	on	the	market	can	add	a	chatbot.	WhatsApp	can	replicate	a	text-based	assistant.	The	technology	layer	is
commodity.	What	is	
not

one	place.	The	agent	bridges	the	data	gap.	The	generative	UI	bridges	the	format	gap.

HOW	IT	WORKS
Every	data	point	in	the	summary	card	is	actionable.	The	

	icon	next	to	medications	means	"Copy	to	Rx	Pad."
Compressed	display	→	Complete	copy:
	the	card	shows	"Glycomet	500	BD"	for	scanning,	but	the	copy	action	sends
the	full	record	(Tab.	Glycomet	500mg,	BD,	after	food,	30	days)	to	the	Medications	section	of	the	Rx	Pad.	Each	data	type
knows	its	routing	target:	symptom	→	Symptoms	section,	medication	→	Medications	section,	diagnosis	→	Diagnosis
section.	In	Point	&	Click	mode,	Copy-to-Rx	fills	the	appropriate	input	field.	In	Voice	mode,	it	adds	to	the	structured
transcript.	The	action	is	always	one	tap.
CHAPTER	05	—	THE	PRIMARY	INTERACTION	LAYER
Canned	Message	State	Machine
How	the	3–4	suggestion	pills	change	across	every	sidebar	tab,	every	Rx	Pad	section,
and	every	doctor	action.	This	is	what	makes	the	agent	discoverable	without	being
intrusive.
CRITICAL	RULE
Input-field-reactive	pills	(S2:	based	on	which	Rx	Pad	section	the	doctor	is	typing	in)	are	active	only	in	Point
&	Click	and	Voice	modes.
	In	Snap	Rx,	Smart	Sync,	and	Drawing	Canvas	modes,	the	system	cannot	detect	which	field
the	doctor	is	interacting	with,	so	pills	are	driven	by	S1	(sidebar	tab),	S3	(available	data),	S4	(patient	state),	and	S5
(time/mode)	only.	The	sidebar	intelligence	and	chat	remain	identical	across	all	5	modes.

THE	DYNAMIC	UI	PRINCIPLE
The	chat	window	never	shows	an	error	like	"No	patient	selected.	Please	select	a	patient	first."	That's	a	system	error,	not	a
solution.	Instead,	the	agent	generates	the	
UI	component	the	doctor	needs
	to	resolve	the	ambiguity	—	an	inline
search,	a	patient	list,	a	confirmation	dialog.	The	response	IS	the	resolution.	This	is	the	core	of	the	AGUI	paradigm:	the
agent	doesn't	just	answer	questions	—	it	generates	the	interface	needed	to	complete	the	task.
